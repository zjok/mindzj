# MindZJ AI 接入与使用操作手册

本文说明如何在 MindZJ 中接入本地离线模型或在线 API Key 模型，并通过 AI 控制笔记软件的常用功能。

## 支持的接入方式

MindZJ 的 AI 设置页提供三类入口：

- LM Studio：适合本机运行的离线模型。
- Ollama：适合本机运行的离线模型。
- 添加 API Key LLM：适合需要 API Key 的在线模型。

本地模型只需要填写接口地址和模型名称，不显示 API Key。在线 API Key 模型只需要在界面里填写模型名称和 API Key；接口地址由应用内部按模型名称使用对应适配器处理，不在设置页展示。

API Key 会保存在系统密钥库中，设置文件只保存模型名称、接入状态等非密钥信息。删除已添加的在线模型时，对应 API Key 会一并清除。

## 接入 LM Studio

1. 启动 LM Studio，并确认本地服务已经开启。
2. 打开 MindZJ，进入“设置 > AI”。
3. 在“本地模型”中选择 `LM Studio`。
4. 接口地址默认是 `http://localhost:1234/v1`。如果本地服务端口不同，改为实际地址。
5. 在“模型名称”中填写 LM Studio 当前加载并对外提供的模型名称。
6. 点击“保存”。
7. 点击“连接”，确认状态变为“已连接”。

## 接入 Ollama

1. 启动 Ollama，并确认本地服务已经开启。
2. 打开 MindZJ，进入“设置 > AI”。
3. 在“本地模型”中选择 `Ollama`。
4. 接口地址默认是 `http://localhost:11434/v1`。如果本地服务地址不同，改为实际地址。
5. 在“模型名称”中填写已拉取并可用的模型名称，例如 `llama3.2`。
6. 点击“保存”。
7. 点击“连接”，确认状态变为“已连接”。

## 添加在线 API Key 模型

1. 打开“设置 > AI”。
2. 点击“添加新模型”，或在提供方列表中选择“添加 API Key LLM”。
3. 填写“模型名称”。
4. 填写服务提供方给出的 API Key。
5. 点击“保存添加”。
6. 保存后，该模型会出现在模型列表中，可直接切换使用。
7. 点击“连接”，确认模型可以响应。

## 删除已添加的在线模型

1. 打开“设置 > AI”。
2. 在“在线模型”中选择已经添加的模型。
3. 点击“删除接入模型”。
4. 确认删除后，MindZJ 会移除该模型配置，并清理对应 API Key。

## 使用 AI 控制笔记

有两种入口：

- 命令面板：打开命令面板，选择“AI 控制 MindZJ”，输入要执行的操作。
- 底部 AI 对话框：按 `Alt+\`` 打开或关闭底部 AI 对话框，输入要执行的操作。

可以输入类似下面的指令：

- 列出当前仓库中的笔记。
- 打开 `diary/2026-04-25.md`。
- 新建 `ideas/local-ai.md`，内容写三条本地 AI 使用建议。
- 搜索包含“项目计划”的笔记。
- 将当前视图切换为阅读模式。
- 修改当前文件，把最后一段改成三条项目行动项。

底部 AI 对话框有一个额外保护：如果指令中没有明确写出具体文件路径，AI 的内容修改只会作用于当前活动文件。需要修改其他文件时，请在指令中写清楚仓库相对路径。

## 可控制的功能范围

AI 可以通过 MindZJ 内置工具执行以下操作：

- 笔记增删改查。
- 文件夹创建和删除。
- 打开笔记、刷新文件树。
- 全文搜索。
- 查询反向链接、正向链接和知识图谱数据。
- 切换当前笔记视图模式。
- 读取或更新应用设置。
- 执行已经注册的插件命令。

## 注意事项

- 删除文件、删除文件夹、覆盖笔记内容等操作会直接修改当前仓库。执行前请在指令中写清楚目标路径和期望结果。
- 本地模型的连接请求由 Tauri 后端代理发送，不经过浏览器跨域预检，因此 LM Studio 和 Ollama 不会再收到无正文的 `OPTIONS` 检查请求。
- 本地模型的工具调用能力取决于所加载模型的能力。较小模型可能更适合简单查询，复杂批量操作建议使用能力更强的模型。
