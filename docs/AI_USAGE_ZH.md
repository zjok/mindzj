# MindZJ AI 接入与使用操作手册

本文说明如何在 MindZJ 中接入本地离线模型或在线 API Key 模型，并通过 AI 命令窗口控制笔记软件。

## 支持的接入方式

MindZJ 的 AI 设置页提供两类模型接入：

- 本地模型：LM Studio、Ollama。
- 在线模型：通过“添加新模型”保存模型名称和 API Key。

本地模型只需要接口地址。点击“测试连接”后，MindZJ 会读取本地服务的 `/models` 列表，并把检测到的模型保存为当前可调用模型。

在线模型不显示接口地址，只填写模型名称和 API Key。API Key 只保存在系统密钥库中；删除已添加模型时，对应 API Key 会一起清除。

## 接入 LM Studio

1. 启动 LM Studio，并确认本地服务已经开启。
2. 打开 MindZJ，进入“设置 > AI”。
3. 在“本地模型”中选择 `LM Studio`。
4. 接口地址默认是 `http://localhost:1234/v1`。如果本地端口不同，改为实际地址。
5. 点击“测试连接”。
6. 状态显示“已连接”后，MindZJ 会使用检测到的模型。

## 接入 Ollama

1. 启动 Ollama，并确认本地服务已经开启。
2. 打开 MindZJ，进入“设置 > AI”。
3. 在“本地模型”中选择 `Ollama`。
4. 接口地址默认是 `http://localhost:11434/v1`。如果本地服务地址不同，改为实际地址。
5. 点击“测试连接”。
6. 状态显示“已连接”后，MindZJ 会使用检测到的模型。

## 添加在线 API Key 模型

1. 打开“设置 > AI”。
2. 点击“添加新模型”。
3. 填写“模型名称”。
4. 填写服务提供方给出的 API Key。可以点击输入框右侧的眼睛图标显示或隐藏内容。
5. 点击“保存添加”。
6. 保存后，该模型会出现在模型列表中，可直接切换使用。
7. 选中该模型后，可点击“测试连接”确认模型可用。

## 删除在线模型

1. 打开“设置 > AI”。
2. 在“在线模型”中选择已经添加的模型。
3. 点击“删除接入模型”。
4. 确认删除后，MindZJ 会移除该模型配置，并清理对应 API Key。

## 使用 AI 命令窗口

有两种入口：

- 按 `Alt+\`` 打开或关闭底部 AI 命令窗口。
- 按 `Ctrl+P` 打开命令面板，选择“打开/关闭 AI命令窗口”。

AI 命令窗口标题会显示当前选中的模型。打开窗口后，编辑区会被推到窗口上方，不会被 AI 命令窗口遮挡。

可以输入类似下面的指令：

- 列出当前仓库中的笔记。
- 打开 `diary/2026-04-25.md`。
- 新建 `ideas/local-ai.md`，内容写三条本地 AI 使用建议。
- 搜索包含“项目计划”的笔记。
- 将当前视图切换为阅读模式。
- 修改当前文件，把最后一段改成三条项目行动项。

底部 AI 命令窗口有额外保护：如果指令中没有明确写出具体文件路径，内容修改只会作用于当前活动文件。需要修改其他文件时，请在指令中写清楚仓库相对路径。

## 可控制的功能范围

AI 可以通过 MindZJ 内置工具执行以下操作：

- 笔记增删改查。
- 文件夹创建和删除。
- 打开笔记、刷新文件树。
- 全文搜索。
- 查询反向链接、正向链接和知识图谱数据。
- 切换当前笔记视图模式。
- 读取或更新应用设置。
- 执行已经注册的插件命令。

## 注意事项

- 删除文件、删除文件夹、覆盖笔记内容等操作会直接修改当前仓库。执行前请在指令中写清楚目标路径和期望结果。
- 本地模型的连接请求由 Tauri 后端代理发送，不经过浏览器跨域预检，因此 LM Studio 和 Ollama 不会收到无正文的 `OPTIONS` 检查请求。
- 本地模型的工具调用能力取决于所加载模型的能力。较小模型可能更适合简单查询，复杂批量操作建议使用能力更强的模型。
