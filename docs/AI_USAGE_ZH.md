# MindZJ AI 接入与使用操作手册

本文说明如何在 MindZJ 中接入本地模型或需要 API Key 的大语言模型，并通过 AI 控制笔记功能。

## 支持的接入方式

MindZJ 的 AI 设置页提供三类入口：

- LM Studio：适合本机运行的离线模型。
- Ollama：适合本机运行的离线模型。
- 添加 API Key LLM：适合需要接口地址和 API Key 的模型服务。

本地模型不需要填写 API Key。需要 API Key 的模型服务会把密钥保存到系统密钥库，设置文件中只保存是否已配置密钥、接口地址和模型名称等非密钥信息。

## 接入 LM Studio

1. 启动 LM Studio，并确认本地服务已经开启。
2. 打开 MindZJ，进入设置页的 AI 页面。
3. 在提供方中选择 `LM Studio`。
4. 接口地址默认是 `http://localhost:1234/v1`，如果你的本地服务端口不同，请改成实际地址。
5. 在模型名称中填写 LM Studio 当前加载并对外提供的模型名称。
6. 点击保存接入模型。
7. 点击测试连接，确认模型可以响应。

## 接入 Ollama

1. 启动 Ollama，并确认本地服务已经开启。
2. 打开 MindZJ，进入设置页的 AI 页面。
3. 在提供方中选择 `Ollama`。
4. 接口地址默认是 `http://localhost:11434/v1`，如果你的本地服务地址不同，请改成实际地址。
5. 在模型名称中填写已拉取并可用的模型名称，例如 `llama3.2`。
6. 点击保存接入模型。
7. 点击测试连接，确认模型可以响应。

## 添加 API Key LLM

1. 打开设置页的 AI 页面。
2. 在提供方中选择 `添加 API Key LLM`。
3. 填写接口地址。不同服务的接口地址不同，请使用服务提供方给出的兼容接口地址。
4. 填写模型名称。
5. 填写 API Key。
6. 点击保存接入模型。
7. 保存后，该模型会出现在提供方下拉列表中，可直接切换使用。
8. 点击测试连接，确认模型可以响应。

## 删除已添加的 API Key LLM

1. 打开设置页的 AI 页面。
2. 在提供方中选择已经添加的模型。
3. 点击删除接入模型。
4. 确认删除后，MindZJ 会移除该模型配置，并清理对应 API Key。

## 使用 AI 控制笔记

1. 确保设置页 AI 页面中已经保存并测试通过一个模型接入。
2. 打开命令面板。
3. 选择 `AI 控制 MindZJ`。
4. 输入你希望 AI 执行的操作。

可以输入类似下面的指令：

- 列出当前仓库中的笔记。
- 打开 `diary/2026-04-25.md`。
- 新建一篇名为 `ideas/local-ai.md` 的笔记，内容写三条本地 AI 使用建议。
- 搜索包含“项目计划”的笔记。
- 将当前视图切换为阅读模式。

AI 会通过 MindZJ 内置的受控工具执行操作，包括笔记增删改查、文件夹创建和删除、全文搜索、打开笔记、链接和图谱查询、刷新文件树、切换视图、读取或更新设置，以及执行已注册的插件命令。

## 注意事项

- 删除文件、删除文件夹、覆盖笔记内容等操作会直接修改当前仓库，请在指令中写清楚目标路径和期望结果。
- 如果模型没有按预期调用工具，先用测试连接确认模型服务可用，再检查接口地址、模型名称和 API Key。
- 本地模型的响应能力取决于你加载的具体模型。较小模型可能更适合简单查询，复杂批量操作建议使用能力更强的模型。
